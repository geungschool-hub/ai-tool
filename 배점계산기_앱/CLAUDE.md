## 1. 이게 뭐 하는 앱인가
한국 고등학교 시험 출제용 점검 도구. 객관식·서답형 문항의 **정답 분포 균형 / 배점 합·목표 / 난이도-배점 정합성**을 실시간 검사하고, **상·중·하 문항 수+총점만 주면 배점을 자동 계산**해 채운다. 여러 시험(버전)을 저장·전환하며 매 학기 재사용하는 게 핵심 목적.

## 2. 코드 지도 (파일 1개, inline `<style>`+`<script>`)
- **전역 상태**
  - `store` = `{ exams:[{id, data, updatedAt}], currentId }` — 다중 시험 보관소
  - `D` = 현재 시험의 `data` (아래 모델)
  - `mcCursor` = 키패드 정답 입력 커서(객관식 행 인덱스)
- **데이터 모델** (`data`): `{ title, mc:[{ans,diff,pts}], sa:[{diff,pts}], targetMC, targetSA, devLimit }`
  - `diff`: **1=하, 2=중, 3=상** (값 의미 고정). `ans`: 1~5, 0=미입력. `pts`: 점수(0=빈칸).
- **저장**: localStorage 키 `munhang_calc_store_v2`. 구버전 단일 키 `munhang_dist_calc`는 첫 실행 시 자동 흡수(마이그레이션).
- **핵심 함수**
  - `loadStore()/persist()/touch()` — 로드/자동저장/수정시각 갱신
  - `renderRows()` `update()` — 표 렌더 / 분포·합계·침범 검사 + 하단 패널 갱신
  - `checkInvade(list)` — 난이도-배점 순서 위반(낮은 난이도가 높은 난이도 배점 침범) 탐지
  - `computeAutoPoints(bands,total,gap)` — 새로 구성 모드 배점 계산
  - `autoPointsKeep(rows,total,gap)` — **입력값 유지** 모드(고정 배점 보존, 빈칸만 채움)
  - `completeDifficulties(rows,a,b,c)` — 빈 난이도를 상/중/하 목표치(deficit)로 채움
  - `autoFill()` — 위 셋을 묶는 진입점(체크박스 `af_keep`로 분기)
  - 문서 전역 `keydown` 핸들러 — 키패드 정답 입력

## 3. 반드시 알아야 할 노하우 / 함정
1. **난이도 값 vs 표시 순서가 다름**: 내부 값은 1/2/3=하/중/상이지만 UI 버튼·밴드는 **상·중·하 순으로 표시**(`DIFF_ORDER=[3,2,1]`). 헷갈리지 말 것.
2. **열 순서**: 번호 → **난이도 → 배점 → 정답** (사용자 지정). 서답형은 난이도 → 배점.
3. **부동소수점 비교 금지**: 배점 합/목표 비교는 항상 `EPS(0.049)` 허용오차 사용. `===` 쓰지 말 것. 내부 계산은 **데시점(×10 정수)** 으로 처리해 오차 차단.
4. **자동 배점 알고리즘 불변식**: 중 기준 `y=(T-gap*(상수-하수))/N` → 반올림 → 잔여를 **하≤중≤상 순서를 깨지 않게** 분배. 결과는 **합계 정확 + 침범 0**. (무작위 2만 케이스 퍼즈로 검증됨.)
5. **`autoPointsKeep` 핵심**: 이미 입력된 배점은 **불변(고정)**, 빈칸만 채움. 각 칸의 동적 상/하한을 `bandMinAbove/bandMaxBelow`로 매번 계산해 **고정값·다른 빈칸과의 순서를 절대 안 깬다**. 순서 제약 때문에 합계를 정확히 못 맞추면 **강제로 깨지 말고** 근사 + 안내 메시지를 반환한다.
6. **`touch()`는 실제 편집에만 호출**: 시험 전환·페이지 로드에서는 호출 금지(안 그러면 모든 시험 수정시각이 열 때마다 갱신됨).
7. **키패드 핸들러 주의**: 전역 keydown이라 `target`이 INPUT/TEXTAREA/SELECT/contenteditable이거나 ctrl/alt/meta면 **반드시 무시**. 안 그러면 배점·제목 타이핑을 가로챈다.
8. **자동 저장**: `update()`마다 `persist()` 호출. 수동 저장 버튼 없음.
9. **펜/필기(canvas) 기능은 존재하지 않는다.** 사용자가 "필기모드 펜"을 말하면 그건 태블릿/OS 화면필기 오버레이지 이 코드가 아니다. 코드에서 찾지 말 것. (정말로 추가 요청이면 그때 캔버스 오버레이를 새로 구현.)

## 4. 사용자 설계 원칙 (강한 피드백)
> 자동화 기능은 **사용자가 일부만 입력해도 나머지를 알아서 완성**해야 한다. "다 입력해야 동작"하게 만들면 의미 없다며 강하게 반발함.

→ 입력 의존 기능은 빠진 값을 합리적 기본/목표로 추론해 완성하고, "먼저 다 채우라"며 중단하지 말 것. 입력된 값은 보존, 빈 곳만 채운다. (`completeDifficulties` + `autoPointsKeep`가 이 원칙의 구현체.)

## 5. 수정·검증 방법
- 문법 검사: `<script>` 블록 추출 후 `node --check`.
- 로직 퍼즈: `document`/`localStorage`를 스텁으로 만들고 스크립트를 eval(초기 실행부 `store = loadStore();...` 이후는 잘라냄). 순수 함수(`computeAutoPoints`,`autoPointsKeep`,`completeDifficulties`,`checkInvade`)를 직접 호출해 **합계 정확·순서 불변·입력 보존**을 검사.
- 변경 후엔 항상 위 두 검사를 돌릴 것.

## 6. 배포
단일 파일·무의존·오프라인. 사용자가 **긍스쿨 허브 개인 허브에 직접 등록** 예정(파일 업로드형이면 HTML만 올리면 됨, 링크형이면 정적 호스팅 후 URL 등록). AI가 외부 플랫폼 로그인·업로드는 불가.
