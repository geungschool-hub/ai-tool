## 긍스쿨 허브에 앱 등록하는 법 (★재사용 절차)
허브 카드는 전부 **URL을 여는 방식** → 로컬 단일 HTML은 먼저 호스팅돼야 등록 가능.

**호스팅 경로 2가지**
- **(A) ai-tool 자체 Pages** ← 가장 간단. `E:\AI tool`(ai-tool repo)는 Pages가 켜져 있어 폴더가 `https://geungschool-hub.github.io/ai-tool/<폴더>/파일.html` 로 바로 서비스됨. 이미 ai-tool에 있는 앱은 **새 레포 없이 이 URL로 즉시 카드 등록 가능**. 공백·한글 경로는 percent-encode(공백=%20). ※ ai-tool은 PUBLIC이라 개인정보·저작권 자료 노출 주의(`git백업_커밋정책.md`).
- **(B) 별도 public 레포 + Pages** → `https://geungschool-hub.github.io/<레포>/`. (배점계산기=score-calc, present-check, todo-geung, random-picker가 이 방식.)
  - 절차: `긍스쿨 허브\<레포>\`에 index.html(+.nojekyll·README) 복사 → `git init/add/commit` → `gh repo create geungschool-hub/<레포> --public --source=. --push` → Pages 켜기 `gh api --method POST repos/geungschool-hub/<레포>/pages -f "source[branch]=main" -f "source[path]=/"` → status:built·curl 200 확인.
  - gh는 `geungschool-hub` 계정 인증됨(repo 권한). `긍스쿨 허브\` 전체는 부모 ai-tool git에서 gitignore(중첩 레포).

**개인 허브(geungschool-personal-hub) 카드 추가 — 암호화 함정 ★**
- 카드 목록은 **브라우저 localStorage에 AES-GCM 암호문**으로만 저장. 비밀번호는 어디에도 저장 안 함(분실=복구불가).
- **AI/터미널은 살아있는 vault를 직접 못 고침. 비번을 받아도 무의미**(브라우저 localStorage 접근 불가, INITIAL_VAULT는 빈 localStorage인 새 기기에서만 적용). → 비번 요구·수령 금지.
- **새 기기용**: 소스 `index.html`의 `SEED.items`에 `{id,title,desc,url,group,emoji,note}` 추가 후 push(첫 설정 때 그 기기 비번으로 자동 암호화).
  - ★단 `INITIAL_VAULT`(배포용 암호화 blob)가 채워져 있으면 **SEED 무시, blob 우선**(storedBlob()이 localStorage→INITIAL_VAULT 순). 2026-07-09 현재 blob 설정됨 → 새 기기 반영은 [현재 기기에서 카드 추가 → ⋯메뉴 '배포용 blob 복사' → INITIAL_VAULT 교체 → push]가 정식 경로. SEED는 blob 비웠을 때의 예비용.
- **현재(이미 설정된) 기기**: 사용자가 직접 — 잠금해제 → ✏편집 → ＋ → 폼 입력 → 저장. (note 필드=관리자 PIN 등 민감정보용, 암호화 저장.)
- 보안 요지: "URL이 비밀이라서"가 아니라 ① 데이터가 내 브라우저에만 있고 ② 비번 암호화 + 5분 자동잠금이라 안전. 모르는 사람이 URL만으로 내 정보 못 봄(남 브라우저는 빈 localStorage→잠금/설정 화면만).

**담임 허브(geungschool-class-hub)**: `index.html`의 `CONTENTS`에 `{title,desc,url,group,emoji,status}` 추가 후 push. **단 등록 여부는 매번 사용자에게 확인**.
