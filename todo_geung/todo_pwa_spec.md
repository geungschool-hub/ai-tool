# 개인 할 일 관리 PWA 개발 사양서

> 본 문서는 Claude Code에 전달할 사양서입니다.
> 기존 출결 관리 PWA와 동일한 패턴(HTML 단일 파일 + localStorage)을 따릅니다.

---

## 1. 목적

고등학교 교사 개인용 할 일 관리 앱. 학생 정보·민감 업무 메모를 외부 SaaS에 입력하지 않고도, 본인 기기 내에서만 안전하게 관리할 수 있도록 한다. 기말 전 다수 업무가 누적된 상황에서 우선순위·주차별 일정·진행 상황을 한눈에 파악하는 것이 핵심 가치다.

## 2. 보안 원칙 (최우선)

1. **모든 데이터는 `localStorage`에만 저장**한다. 외부 서버 통신 일체 금지.
2. 외부 리소스는 **CDN 폰트·아이콘 정도까지만** 허용하고, 분석·트래커는 일체 포함하지 않는다.
3. **데이터 내보내기/가져오기는 JSON 파일 수동 처리** 방식으로만 제공한다 (클라우드 동기화 없음).
4. Service Worker는 캐싱 용도로만 사용하고, 백그라운드 외부 통신은 하지 않는다.

## 3. 기술 스택

- **HTML 단일 파일** (출결 앱과 동일)
- 순수 JavaScript (프레임워크 미사용 권장, 필요 시 경량 라이브러리만)
- CSS는 인라인 또는 단일 `<style>` 블록
- `localStorage` API
- PWA 매니페스트 + 최소 Service Worker (오프라인 동작, 홈 화면 설치 지원)

## 4. 데이터 모델

```javascript
{
  schemaVersion: 1,
  tasks: [
    {
      id: string,           // UUID 또는 timestamp 기반 식별자
      title: string,        // 할 일 제목
      category: string,     // '수업·평가' | '학생기록' | '학급운영'
      priority: number,     // 1 (최우선) ~ 5
      deadline: string|null,// ISO date (YYYY-MM-DD), 없으면 null
      status: string,       // 'todo' | 'doing' | 'done' | 'blocked'
      notes: string,        // 메모 (선택)
      firstStep: string,    // 첫 한 걸음 (선택)
      weekTag: string|null, // 예: '5/27~5/31'
      createdAt: string,    // ISO datetime
      completedAt: string|null
    }
  ],
  settings: {
    darkMode: boolean,
    defaultView: 'priority'|'category'|'week'
  }
}
```

## 5. 기능 요구사항

### 5.1 필수 기능
- 할 일 CRUD (생성·조회·수정·삭제)
- 빠른 추가 입력란 (제목만으로 즉시 추가, 상세는 나중에 편집)
- 체크박스로 완료 처리
- 카테고리·우선순위·마감일·상태별 필터
- 3가지 정렬·그룹 뷰 전환:
  - **우선순위 뷰**: 1순위부터 정렬
  - **카테고리 뷰**: 수업·평가 / 학생기록 / 학급운영 그룹화
  - **주차별 뷰**: weekTag 기준 그룹화 (이번 주 / 다음 주 / 그 이후)
- 완료된 항목은 별도 섹션 (접기 가능)
- 데이터 백업: JSON 파일로 다운로드
- 데이터 복원: JSON 파일 업로드

### 5.2 권장 기능
- 다크 모드 토글
- 마감일 임박 시 시각적 표시 (D-day, 3일 이내 강조 등)
- 항목별 메모 추가 입력
- 검색 기능
- 통계 (오늘/이번 주 완료 건수)

### 5.3 추후 확장 여지
- 알림 기능 (Web Notifications API)
- 캘린더 뷰
- 반복 작업 (매주 월요일 등)

## 6. UI 요구사항

- **모바일 우선** 설계. 데스크탑에서도 사용성 유지.
- 색상은 차분한 톤 (지나치게 화려하지 않게)
- 입력 부담 최소화: 제목만 적고 엔터 → 즉시 등록
- 상세 편집은 항목 탭/클릭 시 모달 또는 인라인 확장
- 카테고리는 색상으로 구분 표시
- 우선순위는 숫자 또는 작은 배지

## 9. 개발 우선순위

1. **MVP**: 5.1 필수 기능 + 초기 데이터 seed + JSON 백업/복원
2. **v1.1**: 다크 모드, 마감일 임박 표시, 검색
3. **v1.2**: 통계, 메모
4. **v2.0**: 알림, 캘린더 뷰

## 10. 파일 구성

단일 HTML 파일 `index.html` 내에 모든 코드를 포함하되, 다음 외부 파일은 별도 생성:

- `manifest.json` (PWA 매니페스트)
- `sw.js` (Service Worker, 캐싱 전용)
- 아이콘 파일 (192x192, 512x512)

배포는 정적 호스팅(예: GitHub Pages) 또는 로컬 실행 모두 가능해야 한다.
