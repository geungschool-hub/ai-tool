## 핵심 동작

- **★선거방(다중 반) 구조 (2026-07-15 밤 개조)**: 다른 반들도 같은 시간에 선거 → 선생님마다 **방**을 만듦(새 방 만들기, 방마다 비밀번호). DB가 `vote/rooms/{방코드}/`로 완전 분리. 학생 URL·QR에 `?r=방코드` 자동 포함. 방 코드+비밀번호로 다른 기기에서도 입장(기존 방 입장 탭). 기기에는 방이 localStorage(`cv.room.v1`)로 기억됨. **다른 선생님께는 admin.html 주소만 공유**하면 됨.
- **교사(admin.html)**: 선거방 입장(첫 입장 기기에서 4단계 진행 안내 팝업 1회, 재열람=개설 화면 하단 "안내") → 선거 개설(선거명·재적인원(선택)·후보 이름들) → 투표 개시
  - **후보가 1명이면 자동으로 찬반 투표(mode:yesno)** — 별도 설정 없음
  - 진행 중: 투표함 SVG + 표 들어올 때마다 투표용지 낙하 연출 + 실시간 투표수 + **QR코드**(학생 URL) + 명부 확인(학번+**투표완료 시각**+✕삭제 버튼)
  - **명부에서 학번 ✕삭제 → 그 학생의 표도 원자적으로 동시 삭제**(open/closed 상태에서만). 삭제된 학생 기기는 실시간으로 완료화면이 풀리고 "다시 투표해 주세요" 안내 후 재투표 가능
  - [투표 마감] → [투표 재개] 가능 → [개표 시작]
  - **개표**: 표를 무작위 순서로 1.3초마다 한 장씩 공개(기호 N번 ○○○ 카드 플립) + **「正」자 집계**. [일시 정지]/[바로 결과]
  - 종료 시 result가 DB에 저장되고 당선 발표 화면(동점이면 "동점 · 재투표", 찬반은 당선/부결/가부동수)
  - [새 선거]로 부회장 선거 이어서 개설. 지난 선거 결과는 개설 화면 하단에 표시
- **학생(index.html)**: 학번 입력(선거인 확인, 이미 투표한 학번 거부) → 투표용지에 기표(빨간 도장 卜/可/否) → 확인 모달 → 투표함 투입 연출
  - **비밀투표(완화됨)**: 표는 `ballots/` push, 명부는 `voters/{학번}={t:시각, b:표키}`. 앱 화면 어디에도 표 내용이 학번과 함께 표시되지 않지만, **표 삭제 기능을 위해 b(표 키) 연결이 DB에 존재** — DB 원본을 열면 연결 추적 가능(사용자 요구 기능상 트레이드오프, 2026-07-15 지시)
  - 중복 방지: voters 트랜잭션(학번당 1회) + localStorage 기기 가드
  - 개표 후에는 학생 화면에도 선거 결과 표시

## 백엔드

- gongju7-vocab RTDB **`vote/`** 최상위 경로(무인증 공개, 표준 레시피). 규칙 정본 `시험목표제출/database.rules.json`에 vote 블록 추가·배포 완료(배포 전 라이브 규칙 대조함 — 로컬 정본과 일치 확인)
- 구조: `vote/rooms/{방코드}/teacherHash`, `.../current`(=진행 중 선거 id), `.../elections/{id}/{title,mode,candidates,total,status,createdAt,voters,ballots,result}`
- status: open → closed(재개 가능) → counting → done

## 관리자 비밀번호

- 방마다 `vote/rooms/{rid}/teacherHash` = SHA-256('classvote.teacher.v1|비밀번호'). 방 생성 때 본인이 설정.
- 변경 UI는 없음 → 바꾸려면 그 방 teacherHash를 REST로 새 해시 PUT(또는 새 방 생성).
- (구버전의 최상위 `vote/teacherHash`·임시 비밀번호는 방 구조 전환으로 폐기·삭제됨.)

## 함정·메모

- 학교망 이 PC 헤드리스/브라우저 테스트에서 `virtual-time-budget` 쓰면 앱의 12초 연결타임아웃이 가상시간으로 즉시 발화해 "연결 실패"가 뜸 — **가짜**. 로컬 파일 스텁에서 RTDB 실읽기 성공 확인함. 최종 확인은 폰에서.
- 좁은 폭(<490px) 헤드리스 스샷 우측 잘림도 가짜(Edge 최소창폭 아티팩트, 520px에서 정상 확인).
- 선거 끝나고 데이터 지우려면 admin 개설 화면의 [선거 데이터 전체 삭제] (teacherHash는 유지됨). **삭제 확인 팝업에 선거별 개표 결과 요약(후보별 득표수)이 떠서 기록을 유도** — 사용자는 투표 직후 학생들 앞에서 바로 삭제하는 운영 방침.
- 앱을 고칠 때 **정본(학급임원선거/)과 배포(class-vote/) 두 폴더가 어긋나지 않게** 둘 다 갱신할 것 (1p1t 이중배포 함정과 동일 구조).
