## 핵심 설계 의도 (코드만 봐선 안 드러나는 것)
- **배점 20점 = 판정 성공 8 + 탐구 보고서(활동지 C섹션) 10 + 실험 과정/안전 2**, 미참여 0점. 점수 옵션 j:[8,5,2,0] r:[10,8,6,4,2,0] s:[2,1,0].
- **부정행위 방지**: 8개 번호병에서 2개 뽑는 조합 28가지(`PAIRS` 배열)를 학생마다 1개씩 배정. 학생은 시료를 "번호"로만 받음.
- **Rh 제외**: 모의 키트 항D 반응 약해 ABO만 평가(항A·항B 혈청만). 같은 ABO형 병이 2개씩(A+/A−) 있을 수 있으나 학생은 번호가 달라 모름.
- **분주 금지**: 빈 용기에 미리 따르면 증발·농도변화로 다음 반 사용 불가 → 스포이트 병째로, 라벨만 번호스티커로 가려 재사용.
- **C-2 채점 원리**: 공혈자=학생 본인 시료라 학생마다 답 다름. O형 환자(혈장 α·β 둘다)→내 시료 A·B·AB면 응집O, O형이면 X / AB형 환자(응집소 없음)→항상 응집X.

## 교사용 앱 동작 & PC 간 이동 (중요)
- 마스터키(번호↔병 매칭) 선택 → 배정표 정답·점수표 자동계산. 🔀 무작위 매칭(반마다 정답 바꿔 재사용). 점수표 합계·평균·최고/최저 자동. 성적 CSV 내보내기.
- **데이터는 localStorage(`bloodTypingTeacherData_v1`)에 저장 = PC/브라우저별.** PC 간 자동 동기화 안 됨 → 백업(JSON)/복원으로만 이동.
- **복원 버튼 2종**:
  - `⬆ 복원(덮어쓰기)` (`importJSON`) — 전체 교체. 기존 반 모두 삭제됨.
  - `➕ 병합 복원(반 추가)` (`mergeJSON`) — 기존 반 유지, 백업의 **새 반만 추가**(동명 반은 건드리지 않음). 2026-06-08 추가.
- **다른 PC의 반과 합칠 때는 반드시 병합 복원 사용.** 예: 집 PC에서 D반 작업 → ⬇ 백업 → 학교 PC에서 ➕ 병합 복원 → C반 유지 + D반 추가.
- 인쇄는 🔀 재배정 **이후**에 해야 정답이 맞음. 학교에서 실험 직전 🔀 다시 누르면 정답 또 바뀌니 주의.
- **📷 학생용 배정표 이미지(정답X)** (`downloadAssignImage`, 2026-06-09 추가) — 툴바 초록 버튼. 정답 열을 뺀 '배정#·이름·시료 번호'만 Canvas로 그려 PNG(`시료배정_{반}.png`) 저장. 외부 라이브러리 없이 오프라인 동작. 칠판 게시·단톡방 공유용. 이름 입력된 행만(없으면 28행 전체 번호만). 🔀 재배정은 정답만 바꾸고 학생-번호쌍 배정은 PAIRS 고정이라 이미지는 언제 뽑아도 동일.

## 주의 / 미해결
- `수행평가 시료 배정.hwp`(C반)는 **27행까지만** 있고 배정#28(①⑤) 누락. 1~27행은 앱 PAIRS와 일치. 교사용 앱이 28쌍 배정표를 동적 생성하므로 hwp는 대체 가능.
- HTML이 PDF보다 최신일 수 있음. **에이전트가 직접 PDF 변환 가능** — Edge 헤드리스:
  `& "C:\Program Files (x86)\Microsoft\Edge\Application\msedge.exe" --headless=new --disable-gpu --no-pdf-header-footer --print-to-pdf="<출력.pdf>" "<입력.html>"`
  (출력의 task_manager ERROR 줄은 무해. `--no-pdf-header-footer`로 머리말/꼬리말 제거. HTML 수정하면 PDF도 다시 뽑을 것.)
