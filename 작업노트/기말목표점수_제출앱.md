# 기말고사 목표 점수 제출 앱 (exam-goal)

## 개요
2학년 기말고사 4일 전, 학생이 **자기가 응시하는 과목을 체크 → 과목별 목표를 적어 제출**하는 시스템.
- 작업 폴더: `시험목표제출/` (index.html=학생, admin.html=교사)
- **배포(주 사용=Firebase Hosting)**: https://exam-goal.web.app/ (학생) · /admin.html (교사)
  - gongju7-vocab 프로젝트의 **별도 호스팅 사이트 'exam-goal'** (multi-site, `firebase hosting:sites:create exam-goal`). firebase.json에 hosting.site="exam-goal", ignore로 html 2개만 배포. 배포: `firebase deploy --only hosting:exam-goal --project gongju7-vocab`
  - 폴백(GitHub Pages): https://geungschool-hub.github.io/exam-goal/ (소스 레포). 단 **학교 와이파이에서 github.io 경로는 인증 실패** 사례 → web.app 사용 권장.
- **★학교망 익명인증 이슈(최종 해결법)**: 학교 와이파이에서 SDK `signInAnonymously()`가 `auth/admin-restricted-operation`으로 거부됨(교사 화면에서 표면화). 진단: 익명가입은 **REST로는 모든 출처(web.app/github.io/no-referer)에서 정상**(토큰 발급 확인)이고 vocab 규칙도 정상 → 백엔드/프로젝트설정 문제 아님. **사용자 브라우저+학교망 환경에서만 SDK 익명로그인이 거부**(서버에서 재현 불가). 
  - **해결**: 익명 인증 의존을 제거. `examGoals` RTDB 규칙을 `.read/.write: true`(공개)로 바꾸고, 두 페이지의 `signInAnonymously()`를 best-effort(실패해도 진행)로 변경. 보안수준은 기존 `auth!=null`과 실질 동일(누구나 익명가입 가능했으므로)·데이터=목표점수(저민감). vocab은 그대로 `auth!=null` 보호(REST로 인증없이 read=401 확인). examGoals 인증없이 write/read=200 확인.
  - 부수: SDK는 영단어와 동일 10.12.2 compat로 통일. Hosting=Firebase web.app(github.io는 폴백). 
  - 교훈: 학교망에서 Firebase 익명인증이 막히면 코드/프로젝트보다 **클라이언트망 문제**일 수 있음 → 저민감 데이터는 공개 규칙+익명인증 제거가 가장 확실한 우회.
  - **★★진짜 근본원인(오래 헤맨 것)**: 교사화면만 `admin-restricted-operation`→`PERMISSION_DENIED`. 알고 보니 **admin.html의 firebaseConfig를 처음 만든 class7-portal-2362b 그대로 두고 gongju7-vocab으로 안 바꿈**(index.html만 바꿨음). class7-portal엔 ①익명인증 꺼짐(→admin-restricted) ②examGoals 공개규칙 없음(→PERMISSION_DENIED). 학생화면(gongju7-vocab)은 정상이라 "교사용에서만 오류"로 나타났던 것. **해결=admin.html config를 gongju7-vocab으로 통일.** 라이브 databaseURL `curl|grep databaseURL`로 확인하는 게 가장 빠른 진단이었음.
  - 교훈: 멀티파일 앱은 **모든 파일의 firebaseConfig가 같은 프로젝트인지** 반드시 확인(특히 index/admin 분리 시). 증상이 "한 화면만" 이상하면 그 화면의 config/databaseURL부터 의심.
  - 부수 변경(원인과 무관하나 유지): examGoals 규칙 공개(`.read/.write:true`, 저민감)+익명로그인 제거(부팅 signOut)+no-cache 헤더. 추후 보안 강화하려면 익명인증+auth!=null로 되돌릴 수 있음(gongju7-vocab 익명인증은 정상).
- 백엔드: **gongju7-vocab** Firebase 프로젝트 재사용(익명 인증 켜져 있음·검증됨), **새 경로 `examGoals/`** 로 영단어 데이터와 완전 분리
  - class7-portal은 학생 익명 인증을 안 써서(교사 이메일/비번만) examGoals에 부적합 → vocab 프로젝트 선택
- 소스 레포: github.com/geungschool-hub/exam-goal (public Pages 전용, 두 HTML만)

## 시험 시간표 보기 (학생화면)
- 학생화면 "📅 기말고사 시간표 보기" 버튼 → 모달. 2학년 시간표 하드코딩(SEED_SCHEDULE, code=과목코드). 교사가 `examGoals/config/schedule` 넣으면 우선.
- 표시: 날짜/요일·교시·시간·과목[코드]·✏️서답형(파란색=서답형포함)·⏰시작시간유의(빨간 시작시간). 출처=2026 1학기 기말 시간표 이미지의 2학년 열만 추출.
- 서답형(✏️) 표기: 6/30 전부, 7/1 대수·역학과에너지, 7/2 생명과학, 7/3 인공지능수학. ⏰: 7/3·7/6 2교시 09:20. (이미지 색 판독이라 교사 확인 권장.)

## 데이터 모델 (RTDB)
- `examGoals/config` : {examTitle, examDate(D-day기준), dueDate, showGrade, intro, subjects:[{id,name}], roster:[{sid,name}]}
- `examGoals/teacherHash` : 교사 비번 SHA-256(salt=examgoal.teacher.v1). **config와 분리**(학생이 읽는 config에 교사해시 노출 방지)
- `examGoals/accounts/{학번}` : {studentId, name, pinHash, createdAt}  ※PIN salt=examgoal.v1.salt
- `examGoals/submissions/{학번}` : {studentId, name, selected:[과목id], subjects:{id:{prev,target,grade,note}}, updatedAt, submittedAt?}
  - note=과목별 "노력할 부분" 짧은 메모(≤60자). 학생 요약카드·교사표(셀 📝 hover title)·CSV(과목명+노력할부분 열) 반영.
  - **하루 학습시간 목표 제거**(요청). 대신 학생화면 하단 건강 메시지 "남은 기간 열심히 하되 건강과 체력에 꼭 유의할 것!".
  - **자동 저장(요청: 제출 안 눌러도 교사가 봄)**: 체크/입력 시 800ms 디바운스로 submissions/{sid} set. `submittedAt`은 학생이 "제출 완료" 눌렀을 때만 기록(자동저장은 기존 submittedAt 보존). → 교사화면 상태=제출완료(submittedAt 有)/작성중(無). 통계·미제출명단·CSV 모두 submittedAt 기준(작성중은 미제출에 ✍️ 표시).
- **iOS 호환**: 앱셸(index) 높이 `100vh`(폴백)→`100dvh`+`-webkit-fill-available`, 스크롤러 min-height:0·overscroll-behavior:contain, safe-area(노치/홈바) padding. 교사 게이트도 vh폴백. (영단어 때 dvh 미지원 구형 iOS에서 빈화면 뜨던 문제 선제 차단.)

## 핵심 동작
- 학생 로그인: 학번(키)+이름 → 신규면 PIN 4자리 생성, 재방문이면 PIN 검증 (영단어 패턴 간소화)
- 과목 선택: 시간표 과목 목록에서 **체크박스로 응시 과목 선택** → 체크하면 그 과목 입력칸(이전점수→목표점수+목표등급) 펼쳐짐
- 하루 학습시간 목표(전체 1개), D-day 카운트다운, 제출 후 "내 목표" 요약카드, 마감 전 수정 가능
- 교사(admin): 비번 게이트(최초 접속 시 설정) / 탭 3개
  - 제출현황: 실시간 표(과목별 목표·▲상승폭·✓신청만··미신청), 통계(제출/평균상승/평균학습시간/미제출), 과목별 신청자수, CSV
  - 시험설정: 제목·시험일·마감일·등급표시·안내문구·**과목목록(코드+과목명) 편집**·비번변경
  - 명단·미제출: 학번+이름 붙여넣기 → 미제출 학생 칩으로 표시

## 교사 최초 세팅 순서
1. admin.html 접속 → 교사 비밀번호 설정(최초 1회)
