# 학습지 빌더 — PRD (Product Requirements Document)

## 1. 배경 및 목적

교사가 직접 학습지를 제작할 수 있는 WYSIWYG 도구. 기존 양식 B 디자인 시스템을 그대로 따르되, HTML 코딩 없이 클릭·타이핑만으로 완성 가능한 편집기를 목표로 한다.

### 제작 배경
- 기존 AI 생성 학습지는 교육적 정확성 검토가 필요해 "참고용"으로만 활용
- 교사가 내용을 직접 작성하고 디자인은 기존 양식을 그대로 유지하는 방식이 최적
- HTML 직접 편집은 코딩 지식 필요 → 도구가 중간 역할 담당

---

## 2. 사용자

- **주 사용자**: 고등학교 교사 1인 (코딩 지식 없음)
- **사용 맥락**: 수업 준비 시 PC 브라우저에서 단독 사용
- **출력물 독자**: 고등학교 2학년 학생

---

## 3. 핵심 요구사항

### 3-1. 편집 방식
- 블록 기반 편집: 페이지에 블록을 추가/삭제/순서 변경
- 모든 텍스트는 클릭 후 직접 입력 (contenteditable)
- 편집 중 실시간으로 최종 출력물 미리보기

### 3-2. 블록 타입
| 블록 | 설명 |
|------|------|
| 섹션 제목 | `.section-title` — 색상 선택 (파랑/초록/주황/빨강) |
| 본문 단락 | 일반 텍스트, 줄바꿈 지원 |
| 빈칸 포함 텍스트 | 텍스트 중간에 빈칸(`.b`) 삽입 가능 — xs/sm/md/lg/xl 크기 선택 |
| 정보 박스 | `.info-box` — 여러 줄 텍스트 |
| 표 | 행/열 수 지정 → 직접 입력. 헤더 색상, 셀 병합(rowspan/colspan) 지원 |
| 이미지 | 파일 업로드 또는 경로 입력. 캡션 옵션 |
| 흐름도 | 박스 + 화살표 가로 나열 (`.flow-chart`) |
| 2열 레이아웃 | 좌/우 두 칸으로 분할 — 각 칸에 독립적으로 블록 배치 가능 |
| 페이지 구분 | 새 `.page` div 삽입 (A4 페이지 분리) |

### 3-3. 페이지 구성
- 각 페이지에 자동으로 헤더(과목·단원·학번/이름) + 푸터(페이지 번호) 포함
- 헤더 정보(과목명, 단원 breadcrumb, 페이지 번호 시작값)는 문서 설정에서 입력

### 3-4. 내보내기
- **학생용 HTML**: 빈칸이 비어있는 상태
- **정답지 HTML**: 빈칸에 정답 표시 (빨간 글씨 `.ans` 스타일)
- 두 버전 모두 기존 학습지와 동일한 HTML 구조 출력

### 3-5. 저장
- 브라우저 localStorage에 자동 저장 (새로고침해도 유지)
- JSON 형식으로 내보내기/불러오기 (백업·공유용)

---

## 4. 비기능 요구사항

- **플랫폼**: 단일 `.html` 파일, 서버 불필요, 브라우저에서 바로 실행
- **의존성**: 외부 CDN 최소화 (오프라인 환경 고려)
- **브라우저**: Chrome 최신 버전 기준 (학교 PC 환경)
- **화면 크기**: 1280px 이상 와이드 레이아웃 (편집 패널 + 미리보기 좌우 분할)

---

## 5. UI 레이아웃

```
┌─────────────────────────────────────────────────────┐
│  상단 툴바: [문서 설정] [블록 추가 ▾] [저장] [내보내기 ▾]  │
├──────────────────┬──────────────────────────────────┤
│                  │                                  │
│   편집 패널       │        미리보기 (양식 B 렌더링)    │
│  (블록 목록,      │                                  │
│   속성 편집)      │         실시간 반영               │
│                  │                                  │
└──────────────────┴──────────────────────────────────┘
```

- 편집 패널: 블록 카드 목록, 드래그로 순서 변경, 카드 클릭 시 속성 편집
- 미리보기: 실제 인쇄 결과와 동일한 양식 B 렌더링

---

## 6. 범위 외 (v1에서 제외)

- 이미지 위 SVG 오버레이 편집 (번호 레이블 직접 배치) → v2 후보
- 교사 외 다른 사용자 지원 (멀티유저, 공유 기능)
- 모바일/태블릿 지원
- 서버 저장/클라우드 동기화

---

## 7. 성공 기준

- 코딩 지식 없이 새 학습지를 처음부터 끝까지 완성 가능
- 출력된 HTML이 기존 학습지와 시각적으로 구분되지 않음
- 학생용/정답지 두 버전을 버튼 한 번으로 생성

---

## 8. 결정 사항

| 항목 | 결정 | 비고 |
|------|------|------|
| 빈칸 정답 입력 시점 | 편집 시 빈칸과 함께 입력 | 내보내기 시 학생용(빈칸)/정답지(빨간 글씨) 분기 |
| 이미지 처리 방식 | base64로 HTML에 포함 | 파일 하나로 완결, 이미지 파일 별도 관리 불필요 |
| 표 셀 병합 (rowspan/colspan) | v1 제외 | UI 구현 복잡도 높음, 단순 표로 대부분 커버 가능 |
| JSON 저장/불러오기 | 포함 | 작업 재개·백업에 필수, localStorage만으로는 부족 |
