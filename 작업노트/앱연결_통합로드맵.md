# 앱 연결성 강화 — 통합 로드맵 (2026-07-10 확정)

한 학기 동안 다루는 명단은 동일 → 앱들이 통합적으로 굴러가게 만드는 방향을 사용자가 확정.
배관 = 생기부 앱의 `#import=<base64(UTF-8 JSON)>` URL 해시 (서버 미전송·공개 repo 안전).

## 3단계 그림
1. **명단 동기화** — 출결앱 백업 JSON을 "명단 마스터"로, 명단 쓰는 교사용 로컬 앱들에 #import= 수신 복제. 학기 초 링크 클릭 몇 번으로 전 앱 명단 맞춤. (생기부 앱은 완료 — `생기부_통합시스템.md` 참고)
2. **기록의 강물을 생기부로** ★가치 최대 — 각 앱의 관찰·채점·세특 재료에 "생기부로 보내기" 버튼 → 학생 번호 키로 생기부 앱 누가기록·참고자료에 병합. 생기부 시즌에 재료가 이미 도착한 상태를 만든다.
3. **공통 백엔드** — 권하지 않음(학교망 차단·개인정보·구조 차이로 비용 대비 효과 낮음). 학생용 앱은 이미 gongju7-vocab 공유로 충분.

## 2단계 첫 배관 완성 (2026-07-10)
**발표체크(진로 심화 탐구 발표) → 생기부 앱** 배포 완료.

### 생기부 앱 수신 확장 (saengbu 커밋 e2dc73a 배포)
- `handleHashImport()`가 payload를 분기: `kind:'records'`면 `importRecords()`, 아니면 기존 명단(classes) 방식 — 두 방식 공존.
- **records payload 규격** (다른 앱에 배관 복제할 때 이대로):
  ```json
  { "v":1, "kind":"records", "source":"발표체크(…)", "className":"진로탐구G", "area":"career",
    "students": { "3": { "name":"김하늘",
      "logs":[{ "date":"2026-06-16", "imp":"g|m|b|null", "text":"관찰 문장" }],
      "paste":"[출처 표시]\n참고자료로 들어갈 텍스트" } } }
  ```
  - `area`: subject / homeroom / autonomy / career / club
  - `imp`: 생기부 인상토글과 동일 키(g=◎인상적 m=○보통 b=–아쉬움)
- 병합 규칙: **추가만, 삭제 없음**. 반 이름 정확히 일치 → 그 반, 없으면 현재 반 폴백(모달에 안내). 번호로 학생 매칭, 명단에 없는 번호는 건너뛰고 결과 alert에 보고. 같은 text의 누가기록·이미 포함된 paste는 자동 중복 차단(재전송 안전).
- 인코딩: `encodeURIComponent(btoa(UTF-8 bytes))` ← 생기부 디코딩(atob→TextDecoder)과 쌍. btoa 전 8192바이트 청크로 String.fromCharCode(스택 오버플로 방지).

### 발표체크 송신 (로컬 전용 앱, `발표체크/index.html`)
- ⋯ 더보기 → **📤 생기부 앱으로 보내기** 모달: 반 이름 입력(기본=현재 분반명, 생기부 앱의 반 이름과 똑같이 맞춰야 함)·영역 선택(기본 진로활동)·[🔗 링크 복사]/[생기부 앱 열기].
- 변환: 축 체크 ◎/– 목록 + 메모 → 누가기록 한 건(date=발표일 6/16·7/7·7/14 매핑, 없으면 오늘 / imp=밴드 상→g 중→m 하→b), 세특 초안 → `[발표체크 세특 초안]` 붙여 paste. 체크·메모·세특 전부 없는 학생은 제외.
- 열기 대상 = https://geungschool-hub.github.io/saengbu/ (사용자가 생기부 데이터를 file:// 로컬에서 쓰고 있으면 [링크 복사] 후 그 브라우저 주소창에 붙여넣기 — localStorage 출처 분리 주의).
