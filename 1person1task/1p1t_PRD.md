# 1인 1역 수행 기록 PRD

## 1. 개요

| 항목 | 내용 |
|------|------|
| 프로젝트명 | 2학년 7반 1인 1역 수행 기록 |
| 목적 | 학생별 1인 1역 활동을 주간 단위로 스스로 기록하고, 담임교사가 한눈에 확인할 수 있는 시스템 |
| 사용 환경 | 교실 전자칠판 / 개인 기기 (태블릿, 스마트폰 등) |
| 구현 형태 | 단일 `.html` 파일 (서버 없음, localStorage 저장) |

---

## 2. 사용자 유형

| 유형 | 진입 방법 | 화면 |
|------|----------|------|
| 학생 | 번호 입력 (예: `07`) | 자신의 역할 기록 화면 |
| 담임교사 | `담임` 입력 → 비밀번호 `0000` | 전체 학생 수행 현황 화면 |

---

## 3. 입장 화면

- 번호 입력란 (숫자 키패드 또는 텍스트 입력)
- `담임` 입력 시 비밀번호 입력 모달 전환
- 잘못된 번호 입력 시 "존재하지 않는 번호입니다" 안내
- 담임 비밀번호 오류 시 "비밀번호가 틀렸습니다" 안내 후 재시도

---

## 4. 학생 기록 화면

### 4-1. 상단 정보
- 학생 이름 + 역할명 표시 (예: `07 박세은 · 학급 회장 공주`)
- 역할 설명 (구체적 활동 내용) 접이식 안내 버튼

### 4-2. 주간 기록표 (핵심)

```
◀  2025년 4월 1주차  ▶

┌──────┬──────┬──────┬──────┬──────┐
│  월  │  화  │  수  │  목  │  금  │
│ 4/7  │ 4/8  │ 4/9  │ 4/10 │ 4/11 │
├──────┼──────┼──────┼──────┼──────┤
│      │      │      │      │      │
│ [기록│ [기록│ [기록│ [기록│ [기록│
│  하기│  하기│  하기│  하기│  하기│
│  버튼│  버튼│  버튼│  버튼│  버튼│
│      │      │      │      │      │
│(입력 │(입력 │(입력 │(입력 │(입력 │
│내용  │내용  │내용  │내용  │내용  │
│표시) │표시) │표시) │표시) │표시) │
└──────┴──────┴──────┴──────┴──────┘
```

- 각 칸에 텍스트 입력 (수행 내용 자유 서술)
- 오늘 날짜 칸 강조 표시
- 이미 기록된 칸: 내용 미리보기 표시, 수정 가능
- 미래 날짜 칸: 흐리게 처리 (입력 불가)
- 주간 이동: 상단 `◀ ▶` 화살표로 이전/다음 주 이동

### 4-3. 하단
- `[저장]` 버튼: 해당 주 기록 일괄 저장
- `[나가기]` 버튼: 입장 화면으로 복귀

---

## 5. 담임교사 화면

### 5-1. 전체 현황 뷰 (기본)

```
◀  2025년 4월 1주차  ▶

┌──────────┬──────────────────────────────────────┐
│ 이름/역할 │  월   │  화   │  수   │  목   │  금  │
├──────────┼───────┼───────┼───────┼───────┼──────┤
│ 박세은   │  ✔   │  ✔   │       │       │      │
│ 학급회장  │(내용) │(내용) │       │       │      │
├──────────┼───────┼───────┼───────┼───────┼──────┤
│ 이가을   │  ✔   │       │  ✔   │       │      │
│ 학급부회장│(내용) │       │(내용) │       │      │
...
```

- 26명 전체 표 형태로 표시
- ✔ 아이콘: 기록 있음 / 빈칸: 미기록
- 셀 클릭 시 해당 학생의 그날 기록 전문 팝업
- 주간 이동 화살표 동일 적용

### 5-2. 학생별 상세 뷰
- 전체 현황 표에서 학생 이름 클릭 → 해당 학생의 전체 기록 이력 모달

### 5-3. 내보내기
- `[CSV 내보내기]`: 현재 주 기록을 엑셀용 CSV로 저장
- `[전체 JSON 백업]` / `[JSON 불러오기]`: 전체 데이터 백업·복원

## 7. 데이터 구조

| 키 | 형식 | 예시 |
|----|------|------|
| `role_[학번]_[YYYY-Www]_[요일]` | 문자열 | `role_20707_2025-W15_mon` → `"학급회의 준비 및 1인 1역 점검"` |

- 주차는 ISO 주차(W15 등) 기준
- 요일: `mon` / `tue` / `wed` / `thu` / `fri`
- 모든 데이터 localStorage 저장

## 8. 기술 사양

| 항목 | 내용 |
|------|------|
| 구현 형태 | 단일 `.html` 파일 |
| 저장 | localStorage |
| **주 사용 환경** | **학생: 스마트폰 (iOS Safari / Android Chrome) → 모바일 우선 설계** |
| 보조 환경 | 담임교사: 전자칠판 또는 PC 브라우저 |
| 뷰포트 | `width=device-width, initial-scale=1, maximum-scale=1` |
| 레이아웃 | 모바일 단일 컬럼 기본, 태블릿/PC는 자연스럽게 확장 |
| 버튼 최소 크기 | 48×48px (모바일 터치 기준) |
| 폰트 최소 크기 | 16px (iOS 자동 확대 방지) |
| 주간표 모바일 처리 | 5칸을 가로 스크롤 또는 카드형으로 표시 |
| 키보드 대응 | textarea 포커스 시 화면 밀림 최소화 |

## 9. 확정 사항

| 항목 | 결정 |
|------|------|
| 기록 수정 제한 | **해당 주차에만 수정 가능** (다른 주차 기록은 읽기 전용) |
| 기록 최소 글자 수 | **제한 없음** (빈 칸도 저장 가능) |

| 양수빈(11번) 역할 | **파티 공주** (손예원, 박효빈, 양수빈 — 3명) |
