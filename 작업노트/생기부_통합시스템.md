### 위계 평가 (📐)
- **데이터**: 반별·영역별 공용 항목 `cur().items[영역키] = [{id,name,sentences[10]}]`, 학생별 선택 `s.areas[영역].levels = {항목id: "1".."10"}`. 백업/병합/복원에 자동 포함(normalizeState·ensureItems·ensureAreas가 구버전 데이터 무손실 마이그레이션).
- **항목 관리 모달**(각 영역 패널의 [⚙ 항목 관리]): 항목 추가/삭제(삭제 시 학생별 levels도 정리·확인모달), 항목명 입력 → [자동 생성]이 활동유형 자동감지(detectActivityType: 탐구/발표/글쓰기/모둠/일반) 템플릿으로 위계 1~10 문장 생성, 문장 개별 수정 가능. 기존 문장 있으면 덮어쓰기 확인.
- **영역→템플릿 매핑**(templatesFor): subject=유형감지, homeroom=행특, career=진로, 자율·동아리=유형감지 후 general이면 자율활동 템플릿.
- **학생 카드**: 항목별 수준 select(옵션에 문장 앞 34자 미리보기) → 문장 즉시 표시(DOM만 갱신, 입력중 텍스트 유실 없음). [⤵ 조합을 초안에 넣기]=선택 문장 join → 초안(기존 초안 있으면 교체 확인모달).
- **AI 연동**: 프롬프트에 `[항목별 평가 수준(위계)]` 블록 + "항목 구분 없이 자연스럽게 연결·통합, 새 사실 추가 금지" 지시. SYSTEM_PROMPT 근거자료에 위계 포함. 생성 가드=누가기록·참고자료·위계 중 하나만 있으면 허용. 카드 점(cdot)도 위계 선택 반영.
