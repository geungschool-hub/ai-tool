# 과학 탐구 보고서 작성법 인터랙티브 웹

## 무엇
- 파일: `보고서작성법_가이드/과학탐구보고서_작성법.html` (단일 HTML, 외부 파일 없음, 오프라인)
- 용도: 1학기 마무리 "주제발표"와 연계해, **과학적 사실을 조사·고찰하여 자기 주장을 논거로 풀어내는 보고서** 쓰는 법을 A~Z로 가르치는 학생용 단계별 웹.
- 대상: 2학년 생명과학(전원 여학생 학급). 학생 어투 = **평어체(~다/~ㄴ다/~이다)로 전부 통일**(존댓말·명령조·문장조각 금지). 중요 단어는 `**단어**`로 감싸면 em()이 형광펜 처리(영역별 색: 핵심 보라·좋은예시 초록·나쁜예시 빨강·팁 주황). 톤 변천(2026-07-07): 해요체→해요+평어 혼용→**최종 평어체 통일**(혼용은 "이상한 사람 같다"는 피드백). AI 채팅 답변은 계속 공손.
- 제출/PDF: 계획서 단계에 **학번·이름 입력 + PDF 내보내기(window.print, #printdoc + @media print, 학번·이름·작성일 자동)**. 제출 **3옵션**(①계획서 ②보고서 ③발표, 셋 다 세특 반영). 발표 안 해도 세특 작성 가능하게 한 의도. ※ **①계획서 제출은 이메일→Firebase 원클릭으로 변경됨(2026-07-07, 아래 "계획서 원클릭 제출" 참고). 이메일 안내는 제거됨.** PDF는 오프라인 폴백.

## 왜 이 방향 (사용자 진단)
- 학생들의 실제 약점: **논거(근거) 개념 부재, 주장·사실 미구분, 횡설수설, 장문 두려움, 사실 나열만 하고 고찰(해석) 못 함.**
- 그래서 "자료 조사·정리"가 아니라 **주장–논거–고찰**을 중심축으로 재설계함. (자료 조사 단계는 "논거 모으기"로 유지)

## 구성 (화면 11개)
1. 🎯 주제와 탐구 질문 정하기 / 2. ⚖️ 주장과 논거 이해하기(핵심 개념) / 3. 🔎 논거 모으기(자료 조사) /
4. 🗂️ 개요로 뼈대 잡기 / 5. ✍️ 문단으로 쌓아 쓰기(횡설수설·장문 대응) / 6. 🔬 과학적으로 고찰하기 / 7. 🔍 서론·결론 쓰고 다듬기
- \+ 📌 인용·출처 표기법(자료 5종 형식+예시, 표절 방지) / 📋 보고서 구조 / ✅ 자기점검 체크리스트(13항목·주장-논거-고찰 반영) / 📝 내 탐구 계획서(구 "내 보고서 계획서")
- 각 단계 = keyIdea 배너 + 설명 + **좋은👍/나쁜👎 예시 대비** + 꿀팁 + ✏️직접 해보기(입력→localStorage 자동저장)
- 계획서 화면 = 단계별 입력을 취합 → **전체 복사** 버튼(실제 보고서 뼈대로 사용) / 전체 지우기

## 기술·제작 메모
- 콘텐츠는 워크플로(초안 medium→검증 high, 18에이전트)로 생성. 예시 소재는 카페인/미세먼지/가공육 등 과학 주제.
- 데이터는 `<script type="application/json" id="report-data">`에 넣고 JSON.parse로 렌더(따옴표 깨짐 방지). 로직은 데이터-드리븐 렌더.
- 앱셸 100dvh·본문만 스크롤(iOS 먹통 방지), 44px 버튼·16px+ 폰트, 좌우 화살표 이동, 하단 제작자 문구 포함.
- 검증: 임베드 JSON 파싱 + 인라인 JS `node --check` + Edge 헤드리스 스샷(1단계/체크리스트/계획서) 확인 완료.
